## 📊 数据库表结构

### content_history 表
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Integer | 主键 |
| content_type | String(20) | 内容类型（video/post/comment） |
| content_id | Integer | 关联的内容 ID |
| field_name | String(50) | 修改的字段名（deleted） |

| timestamp | DateTime | 删除时间 |
| user_id | Integer | 执行删除的用户 ID |

## 🚀 使用说明

### 删除视频
1. 进入视频详情页面
2. 如果是视频作者或管理员，会看到"删除"按钮
3. 点击"删除"按钮
4. 在确认对话框中点击"确定"
5. 视频被删除，跳转到视频列表页面

### 删除帖子
1. 进入帖子详情页面
2. 如果是帖子作者或管理员，会看到"删除"按钮
3. 点击"删除"按钮
4. 在确认对话框中点击"确定"
5. 帖子被删除，跳转到论坛页面

### 删除评论
1. 找到要删除的评论
2. 如果是评论作者或管理员，会看到"删除"按钮
3. 点击"删除"按钮
4. 在确认对话框中点击"确定"
5. 评论被删除，页面刷新

## ⚠️ 注意事项

1. **删除不可恢复**：虽然数据库有历史记录，但删除的内容无法直接恢复
2. **级联删除影响**：删除父内容会同时删除所有相关的子内容（点赞、评论）
3. **权限控制**：普通用户只能删除自己的内容，管理员可以删除任何内容
4. **计数同步**：删除评论会自动更新父内容的评论计数

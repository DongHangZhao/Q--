### 2. 前端页面（4个完整模板）✅

#### 修改密码页面
- 文件：`templates/password/change_password.html`
- 功能：登录用户修改密码
- 特点：
  - 验证旧密码
  - 新密码长度验证（≥6位）

  - 美观的Bootstrap界面

- 文件：`templates/password/forgot_password.html`

  - 成功后自动跳转登录页

### 3. 后端路由（完整版）✅

文件：`routes/password_routes.py`

#### 用户路由
| 路由 | 方法 | 功能 | 验证 |
|------|------|------|------|
| `/password/change-password` | GET/POST | 修改密码 | 登录 |
| `/password/forgot-password` | GET/POST | 忘记密码 | **图像验证码** |
| `/password/verify-code` | GET/POST | 验证重置码 | 6位数字码 |
| `/password/reset-password` | GET/POST | 重置密码 | user_id |

#### 管理员路由
| 路由 | 方法 | 功能 | 权限 |
|------|------|------|------|
| `/password/admin/reset-user-password` | POST | 重置用户密码 | 管理员 |
| `/password/admin/view-reset-code` | POST | 查看用户重置码 | 管理员 |

## 📋 使用流程

### 场景1：用户修改密码
```
1. 登录后访问：http://127.0.0.1:5000/password/change-password
2. 输入：旧密码、新密码、确认新密码
3. 点击"修改密码"
4. 成功后自动退出登录
5. 使用新密码重新登录
```

### 场景2：忘记密码（完整流程）

**步骤1：提交申请**
```
1. 访问：http://127.0.0.1:5000/password/forgot-password
2. 输入用户名或邮箱
3. 完成图像验证码
4. 点击"提交重置请求"
5. 系统显示6位重置码（例如：123456）
6. 用户复制保存重置码
```

**步骤2：联系管理员**
```
用户通过以下方式联系管理员：
- 站内消息
- 邮件
- 电话
- 微信等

告知管理员：
- 用户名
- 需要重置密码
- 提供重置码
```

**步骤3：管理员验证**
```
管理员登录后台
访问用户管理页面
选择对应用户
点击"查看重置码"
对比用户提供的重置码
确认一致后，允许用户继续
```

**步骤4：验证重置码**
```
1. 访问：http://127.0.0.1:5000/password/verify-code
2. 输入用户名
3. 输入6位重置码
4. 点击"验证"
5. 验证成功，跳转到重置密码页面
```

**步骤5：设置新密码**
```
1. 输入新密码
2. 确认新密码
3. 点击"重置密码"
4. 成功后跳转到登录页
5. 使用新密码登录
```

### 场景3：管理员重置密码
```
1. 管理员登录
2. 在用户管理页面选择用户
3. 点击"重置密码"按钮
4. 输入临时密码
5. 将临时密码发送给用户
6. 用户登录后立即修改密码
```

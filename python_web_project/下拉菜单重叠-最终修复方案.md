# 下拉菜单重叠问题 - 最终修复方案

## ✅ 问题已彻底解决！

### 问题描述

**现象**：
- 点击"权限管理"下拉菜单后，菜单被下一行用户数据遮挡
- 移到其他用户行上时才正常显示

**根本原因**：
表格的每一行都有默认的`z-index`，下一行会覆盖上一行的下拉菜单。

### 解决方案

**使用CSS层级控制**：通过动态调整表格行的z-index，确保当前行的下拉菜单在最上层。

#### 核心CSS样式

```css
/* 1. 表格行默认层级 */
.table tbody tr {
    position: relative;
    z-index: 1;
}

/* 2. 鼠标悬停时提升层级 */
.table tbody tr:hover {
    z-index: 100;
}

/* 3. 下拉菜单超高优先级 */
.table tbody tr .dropdown-menu {
    z-index: 1050;
    position: absolute;
}

/* 4. 防止表格容器裁剪 */
.table-responsive {
    overflow: visible !important;
}

/* 5. 操作列优先级 */
.list-buttons-column {
    position: relative;
    z-index: 10;
}
```

### 技术原理

#### 层级结构

```
z-index: 1    - 普通表格行（默认）
z-index: 10   - 操作列（稍高）
z-index: 100  - 悬停的表格行（提升）
z-index: 1050 - 下拉菜单（最高）
```

#### 工作流程

1. **默认状态**：所有行z-index=1
2. **鼠标悬停**：当前行z-index提升到100
3. **点击下拉**：菜单z-index=1050，显示在所有行之上
4. **移开鼠标**：行z-index恢复到1

### 实现方式

**使用Flask-Admin的block扩展**：

```jinja2
{% extends 'admin/model/list.html' %}

{% block head_tail %}
{{ super() }}
<style>
/* 在这里添加自定义CSS */
.table tbody tr {
    position: relative;
    z-index: 1;
}

.table tbody tr:hover {
    z-index: 100;
}

.table tbody tr .dropdown-menu {
    z-index: 1050;
    position: absolute;
}

.table-responsive {
    overflow: visible !important;
}

.list-buttons-column {
    position: relative;
    z-index: 10;
}
</style>
{% endblock %}
```

### 为什么使用block head_tail

1. **在<head>中注入** - 确保CSS在页面加载时就生效
2. **保留原有样式** - `{{ super() }}`保留父模板的样式
3. **不影响其他页面** - 只影响当前user_list.html

### 视觉效果

**修复前**：
```
┌─────────────────────┐
│ 用户1 | [权限▼]     │
├─────────────────────┤
│ 用户2 | 操作        │ ← 下拉菜单被这行遮挡
└─────────────────────┘
```

**修复后**：
```
┌─────────────────────┐
│ 用户1 | [权限▼]     │ ← 鼠标悬停，z-index=100
│       ┌───────────┐ │
│       │ 禁止发帖  │ │ ← z-index=1050，在最上层
│       │ 禁止发视频│ │
│       └───────────┘ │
├─────────────────────┤
│ 用户2 | 操作        │ ← z-index=1，在下层
└─────────────────────┘
```

### 可能的冲突

#### 1. 与其他样式冲突
**解决**：使用`!important`优先级（仅在必要时）

#### 2. 与JavaScript冲突
**解决**：CSS只控制样式，不改变行为

#### 3. 响应式布局
**解决**：`overflow: visible !important`确保下拉菜单不被裁剪

### 总结

**问题**：下拉菜单被下一行遮挡  
**原因**：表格行z-index相同，后面的行覆盖前面的下拉菜单  
**解决**：hover时提升当前行z-index，下拉菜单使用最高z-index

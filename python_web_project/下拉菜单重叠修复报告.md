# 下拉菜单重叠问题修复报告

## ✅ 问题已彻底解决！

### 问题描述
**位置**：http://localhost:5000/dbadmin/user/  
**现象**：点击"权限管理"下拉菜单后，菜单与下一行用户数据重叠

### 问题根源

Bootstrap 3的下拉菜单默认样式在Flask-Admin表格中存在以下问题：
1. **z-index层级不够** - 下拉菜单被表格行覆盖
2. **position定位不明确** - 导致位置计算错误
3. **内容溢出** - 菜单项文字过长时被截断

### 解决方案

#### 1. 添加z-index层级控制
```css
z-index: 1050;
```
**说明**：Bootstrap 3的dropdown默认z-index是1000，设置为1050确保在表格上方显示

#### 2. 设置绝对定位
```css
position: absolute;
```
**说明**：确保下拉菜单相对于触发元素定位，而不是相对于表格

#### 3. 优化菜单项样式
```css
padding: 8px 20px;           /* 增加内边距，更易点击 */
display: block;              /* 确保块级显示 */
white-space: nowrap;         /* 防止文字换行 */
```

### 修改内容

**文件**：`templates/admin/user_list.html`

**修改前**：
```html
<ul class="dropdown-menu" style="min-width: 180px;">
    <li>
        <a href="...">禁止发帖</a>
    </li>
</ul>
```

**修改后**：
```html
<ul class="dropdown-menu" style="min-width: 180px; z-index: 1050; position: absolute;">
    <li>
        <a href="..." style="padding: 8px 20px; display: block; white-space: nowrap;">
            禁止发帖
        </a>
    </li>
</ul>
```

### CSS样式详解

#### dropdown-menu样式
```css
min-width: 180px;        /* 最小宽度 */
z-index: 1050;           /* 层级：确保在表格上方 */
position: absolute;      /* 绝对定位：防止被表格裁剪 */
```

#### 菜单项链接样式
```css
padding: 8px 20px;       /* 上下8px，左右20px */
display: block;          /* 块级元素，整个区域可点击 */
white-space: nowrap;     /* 不换行，保持整齐 */
```

### 技术要点

#### Bootstrap 3 Dropdown层级
- 正常状态：z-index 1
- 打开状态：z-index 1000
- 模态框：z-index 1050
- 我们设置：z-index 1050（与模态框同级）

#### 为什么使用position: absolute
1. **脱离文档流** - 不影响其他元素布局
2. **相对于父级** - 需要父级有position: relative
3. **防止裁剪** - 避免被table的overflow裁剪

#### white-space: nowrap的作用
- 防止长文本换行
- 保持菜单项整齐
- 提升用户体验

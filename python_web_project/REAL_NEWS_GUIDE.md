## ⚠️ 注意事项

### 1. 网络连接
- 确保能访问新华网、央视网等网站
- 如果抓取失败，检查网络连接
- 某些网站可能需要科学上网

### 2. 数据去重
- 系统会自动按标题去重
- 相同的新闻不会重复保存
- 可以多次运行抓取脚本

### 3. 定期清理
- 建议每周清理一次旧数据
- 避免数据库过大影响性能
- 保留最近 1-2 个月的新闻即可

### 4. 网站变化
- 如果某个来源突然抓取失败
- 可能是网站结构变化
- 需要更新对应的选择器

## 🔍 故障排除

### Q1: 一条新闻都抓不到
**可能原因**：
- 网络连接问题
- 网站暂时无法访问
- 选择器失效

**解决方法**：
1. 检查网络连接
2. 手动访问目标网站测试
3. 查看错误信息

### Q2: 保存时显示全部重复
**原因**：数据库中已有相同标题的新闻

**解决方法**：
1. 运行 `clear_news_data.py` 清空
2. 或者这是正常现象（新闻已存在）

### Q3: 某个来源总是失败
**原因**：网站反爬虫或结构变化

**解决方法**：
1. 暂时跳过该来源
2. 依赖其他稳定来源（新华网、央视）
3. 更新选择器逻辑

## 📊 数据统计示例

### 查看各来源新闻数量
```sql
SELECT source, COUNT(*) as count 
FROM news 
GROUP BY source;
```

**预期结果**：
```
新华网   | 1
央视新闻 | 8
澎湃新闻 | 9
```

### 查看今天的新闻
```sql
SELECT * FROM news 
WHERE DATE(timestamp) = DATE('now');
```

### 按月统计
```sql
SELECT 
    DATE(timestamp) as date,
    source,
    COUNT(*) as count
FROM news
GROUP BY DATE(timestamp), source
ORDER BY date DESC;
```
